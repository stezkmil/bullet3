Current implementation state

1. Anchor solver / deformable coupling
- src/BulletSoftBody/btDeformableBackwardEulerObjective.cpp
  - dynamic anchored nodes are detected with isDynamicRigidAnchorNode(...)
  - explicit-force application for implicit solve skips dynamic anchored nodes
  - their effectiveMass_inv row is zeroed for that explicit application path
  - a very small anchor reaction transfer was added:
    - transferDynamicAnchorReactionImpulses(...)
    - this feeds a tiny body-averaged fraction of assembled anchor-node force back to the rigid body
    - currently scaled by 0.02 / anchorCountForBody and additionally clamped
  - this was introduced because full-force transfer exploded badly in an earlier experiment

2. Anchor state seeding / solver staging
- src/BulletSoftBody/btDeformableBodySolver.cpp
  - seedDynamicAnchorTargets(...) now only seeds target q from rigid target position/velocity
  - it no longer overwrites node velocity before solveConstraint()
  - this change was important because pre-forcing node velocity made solveConstraint residual almost zero, so the anchor solver had no real work to do
  - applyTransforms(...) still projects anchor position to rigid anchorWorld at end of step, but does not overwrite solved velocity anymore

3. Anchor constraint solve
- src/BulletSoftBody/btDeformableContactConstraint.cpp
  - solveConstraint():
    - still applies full rigid-side impulse via rigidCol->applyImpulse(impulse, m_anchor->m_c1)
    - stores convergence-based relaxation state back into the anchor struct
  - solveSplitImpulse():
    - uses vb = node->m_v + node->m_splitv
    - predicts soft anchor position with x + dt * vb
    - split diagnostics still active

Important experiments tried today

1. Hard kinematic resync of dynamic anchors
- Stabilized spikes very well
- But made the anchored end too prescribed and killed believable spring-back

2. Full explicit-force transfer from anchor nodes to rigid body
- Exploded immediately
- Produced absurd rigid motion and giant impulses
- Replaced with the much smaller transfer currently in tree

3. Implicit timestep comparison
- 0.002 implicit + projection was still pathological
- 0.001 implicit + projection was much better
- This strongly suggests timestep matters, but it is not the whole problem

4. COM-based rigid picking
- Helped a lot with “cube only rotates under mouse”

5. Scripted grab trace

Best current hypothesis
- We fixed the anchor spike bug and improved interaction tooling, but the main physical issue remains:
  dynamic anchors are still not modeled as true constrained DOFs with consistent two-way coupling inside the implicit solve.
- The remaining S-shape / weak spring-back likely comes from the anchored boundary still being handled by a hybrid of:
  - partial boundary-condition behavior
  - separate velocity matching
  - end-of-step positional projection
  rather than a single coherent constrained solve.
